# Live-Commerce Script Framework

Use this as the neutral structure before applying a specific streamer's persona.

## Standard Flow

1. Retention: call newcomers, preview benefit, identify the target audience.
2. Pain point: name the user's concrete problem or scene.
3. Product seeding: connect feature to benefit.
4. Demonstration: show try-on, use, comparison, texture, size, or scenario proof.
5. Trust: explain expertise, material, origin, after-sales, or user feedback when factual.
6. Price / welfare: state the real offer and required confirmation facts.
7. Conversion: guide cart/link/order action without fake pressure.
8. Interaction: answer objections and route users by size/spec/need.
9. Transition: summarize the current SKU and preview the next one.

## Evaluation Rubric Gates

Before presenting a generated script, check `references/script_evaluation_rubric.md`.

- P0 gates: instruction following, factual boundary, product-audience fit, scenarioized selling point, information richness, compliance boundary, persona fit, category atmosphere.
- P1 upgrades: fluency, content extension, empathy/atmosphere, conversion ability.
- Scenarioized selling point: include usage scene, action/process, comparison reference, and perceivable detail.
- Information richness: cover at least four categories when available: display, parameters, fit recommendation, usage instructions, proof, after-sales.
- Conversion ability: include a clear action path plus at least two conversion information types.
- Facts: price, stock, gifts, official policies, proof, rankings, and endorsements must be user-provided or marked `需确认`.

## Output Families

- Full teleprompter script.
- Short loop script.
